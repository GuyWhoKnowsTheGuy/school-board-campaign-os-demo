# Dashboard Specification

## Experience goal

The dashboard should feel like Mission Control for a school board campaign: calm, direct, and operationally useful.

## Required panels

1. **Campaign Readiness** — scores, blockers, not-ready warnings.
2. **Next Actions** — earliest/highest-risk tasks.
3. **Election Mechanics** — deadline estimates, petition packet, filing checklist, source links.
4. **District Intelligence** — CCSD 46 schools, board, strategic signals, data-source gaps.
5. **Political Terrain** — tribal knowledge warnings, power map, risks.
6. **Campaign Operations** — module checklists for volunteers, fundraising, messaging, digital, endorsements, forums, GOTV.
7. **Michelle Validation** — interview/review process for converting candidate experience into reusable knowledge.
8. **Postmortem Loop** — structured lesson capture.

## Interaction requirements in v0

- Candidate readiness questions are interactive.
- Scores update immediately.
- Red flags are surfaced clearly.
- Tasks can be checked off and persist in localStorage.
- Sections are navigable without needing separate docs.
- All source-backed claims link to source index.

## Future UX requirements

- District picker
- Election-cycle generator
- Candidate role-based onboarding
- Import/export campaign workspace
- Postmortem submission workflow
- Evidence heatmap for source confidence
- Public/private mode toggle
